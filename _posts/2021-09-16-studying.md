## 1. HAVING 절 
#### 특징
* 1) 정의 : WHERE 절과 비슷하지만 <U>**그룹을 나타내는 결과집합 행의 조건**</U>이된다.
* 2) HAVING 절과 WHERE절의 순서가 바뀌어도 결과물이 같다.
* 3) 하지만, 쿼리문 수행시 GROUP BY 먼저 적용후 HAVING절이 수행되므로 절차적으론 GROUP BY 먼저 쓰는게 맞다.
* 4) WHERE절 +  GROUP BY 사용 :  GROUP BY 사용절 조전 적용하므로 효율적이다. 반면에, GROUP BY + HAVING절은 연산 후 조건적용되기때문에 비효율적이다.
* 4) HAVING절의 조건 변경은 출력된 레코드의 개수만 변경가능하다. 데이터 자체를 변경할 순 없음.

## 2. CASE 표현식
#### 공부하면서 아리송했던것들 정리
* 1) DECODE 함수 : IF ELSE 함수와 동일 EX) DECODE('GENDER', 'M', '남자', 'F', '여자', '기타') => GENDER는 컬럼명, M은 조건 남자는 결과이다. 마지막 '기타'는 ELSE일때 사용.

## 3. 집계함수와 NULL 처리
#### 공부하면서 아리송했던것들 정리
* 1) AVG는 NULL을 제외후 계산한다.
* 2) 집계합수 계산시 NULL은 그냥 두면 NULL을 빼고 계산한다, 괜히 NULL을 0으로 변화하면 0까지 넣어서 계산하므로 부하가 올라간다.

## 4. ORDER BY 절
#### 정리
* 1) 정의 : 다양한 목적에 맞게 특정 칼럼을 기준으로 정렬, 출력하는데 사용한다.
* 2) ALIAS 명이사 순서 정수로도 ORDER BY절 정의 가능하다.
* 3) 기본적으로 오름차순으로 사용됨.
* 4) ORACLE에서는 NULL을 가장 큰값으로 간주해서 오름차순시에는 맨뒤로, 내림차순시에는 맨아래로 정렬한다.
