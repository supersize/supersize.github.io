## 2. 문자유형비교 방법 중 난해한 것들 
* 비교 연산자의 양쪽이 모두 char 유형인 경우
  > 1. 길이가 서로 다르면, 작은 쪽에 스페이스를 추가해 길이를 같게한 후 비교한다. 
  > 2. 서로 다른 글자가 나올 때까지 비교한다. 
  > 3. 달라진 첫 번째 문자의 값에 따라 크기를 결정한다 --> 뭔소리(?)
  > 4. 문자 끝 블랭크 수만 다르다면 서로 같은 값으로 결정한다. --> 뭔소리(?)
  
* 비교 연산자의 어느 한쪽이 모두 varchar 유형인 경우
  > 1. 서로 다른 문자가 나올 때까지 비교한다. 
  > 2. 길이가 서로 다르면, 짧은 쪽이 끝날때 까지만 비교한 후 길이가 깉 것이 크다고 판단한다.
  > 3. 길이가 같고 달라진 것이 없다면 같다고 판단한다.
  > 4. varchar는 not null 까지의 길이를 말한다. 즉, 문자 끝 블랭크도 문자로 취급한다.
  
## 3. 숫자타입의 문자열 비교
* 숫자 유형 칼럼의 경우 숫자로 변환 가능한 문자열과 비교되면 상대 타입을 숫자타입으로 바꾸어 비교한다.
 > where height >= 170 조건을 where height >= '170'이로고 표현해도 내부적으로 '170'이라는 문자열을 숫자 유형으로 바꿔 처리한다.

## 4. 와일드 카드
* '%' 
* : 0개 이상의 어떤 문자를 의미한다.

* '_'
*  : 1개인 단일 문자를 의미한다.

## 4. 연산관련 NULL 특성
> 1. NULL 값과의 수치연산은 NULL 값을 리턴한다.
> 2. NULL 값과의 비교연산은 거짓(FALSE)를 리턴한다.
> 3. 어떤 값과 비교할 수도 없으며, 특정 값보다 크다/작다고 표현할 수 없다.

## 5. 논리연산자 
* 우선순위
* : 우선순이는 (), NOT, AND, OR 순이다.

* IN 조건과 OR 조건
*  : IN과 OR 조건은 결과도 같고 내부적으로 처리하는 방식도 같다.

## 6. GROUP BY 함수
> 1. NULL 값을 가진 행을 제외하고 수행한다.
> 2. WHERE절에는 올수 없다.
> 3. WHEER절을 전체 데이터를 GROUP으로 나누기 전에 미리 행들을 제거한다.
